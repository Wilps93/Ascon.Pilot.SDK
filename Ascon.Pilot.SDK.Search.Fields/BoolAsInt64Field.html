Класс BoolAsInt64Field

Описание

Класс для работы с булевыми полями, хранящимися в виде 64-битных целых чисел (0L и 1L) в поисковых запросах. Наследуется от BoolField.

Наследование

  • System.Object
  • BoolField
  • BoolAsInt64Field

Конструктор


internal BoolAsInt64Field(string fieldName)

Инициализирует новый экземпляр класса с указанием имени поля.

Методы

Be(bool value)


public override ISearchTerm Be(bool value)

Создает поисковый термин для булева значения, преобразованного в 64-битное целое число (0L или 1L).

WithPrefix(long value)


private string WithPrefix(long value)

Добавляет специальный префикс для форматирования 64-битных числовых значений в поисковом запросе.

Особенности реализации

  • Булевы значения преобразуются: false → 0L, true → 1L
  • Префикс "@" (символ @) указывает на 64-битный числовой формат
  • Используется для полей, требующих хранения логических значений в расширенном числовом формате

Пример использования


// Создание условия поиска для 64-битного булева поля
var field = new BoolAsInt64Field("IsArchived");
ISearchTerm term = field.Be(true);

// Результат: условие "IsArchived: @1"
Важно: Класс имеет внутренний модификатор доступа (internal) и предназначен исключительно для внутреннего использования в инфраструктуре SDK. Рекомендуется использовать фабричные методы поисковой системы для создания условий.
Рекомендации:
  • Проверяйте требования поисковой системы к формату числовых данных
  • Для 32-битных значений используйте BoolAsInt32Field
