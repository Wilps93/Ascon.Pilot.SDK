Перечисление TriggerType

Общее описание

Перечисление TriggerType определяет типы триггеров, инициирующих выполнение автоматизированных процессов. Используется для:

  • Указания контекста запуска активностей
  • Ограничения области применения автоматизации
  • Определения источника событий (клиент/сервер)

Ключевой элемент конфигурации автоматизированных рабочих процессов.

Значения перечисления

Значение Описание
None Триггер не определен (автоматика не срабатывает)
Client Срабатывание только при клиентских событиях (GUI-действия)
Server Срабатывание только при серверных событиях (бэкенд-логика)
ServerAndClient Срабатывание как на клиентских, так и на серверных событиях

Примеры использования

Проверка типа триггера


public async Task RunAsync(..., TriggerType triggerType)
{
    if (triggerType.HasFlag(TriggerType.Client))
    {
        // Логика для клиентских событий
    }
    
    if (triggerType.HasFlag(TriggerType.Server))
    {
        // Логика для серверных событий
    }
}

Конфигурация активности


var activityParams = new Dictionary<string, object>
{
    { AutomationDefaults.TriggerType, TriggerType.ServerAndClient }
};
Важно:
  • Значение ServerAndClient включает оба типа триггеров
  • None используется для деактивации автоматики
  • Учитывайте производительность при использовании серверных триггеров
  • Клиентские триггеры могут требовать дополнительной визуализации
