Класс Stage [Устаревший]

Описание

Представляет этап в шаблоне задачи. Класс признан устаревшим - рекомендуется использовать современные альтернативы для новых разработок.

Содержит список исполнителей и метаданные этапа.

Свойства

Свойство Тип Описание
Executors List Список исполнителей этапа (инициализируется в конструкторе)
Kind TaskKind Тип задачи этапа
Version int Версия этапа

Методы

Переопределенные методы

Equals(Stage) Сравнивает текущий объект с другим этапом
Equals(object) Сравнивает с произвольным объектом
GetHashCode() Возвращает хеш-код объекта

Примеры использования


// Создание этапа
var stage = new Stage
{
    Kind = TaskKind.Approval,
    Version = 1,
    Executors = 
    {
        new Executor { Role = "manager", IsDeadlineSet = true }
    }
};

// Сравнение этапов
Stage anotherStage = GetStage();
bool areEqual = stage.Equals(anotherStage);

// Добавление исполнителя
stage.Executors.Add(new Executor { Role = "auditor" });
Важно:
  • Класс помечен как [Obsolete] - использование в новых проектах не рекомендуется
  • Сравнение этапов происходит по всем исполнителям
  • Хеш-код зависит от всех элементов списка Executors
  • Используемые классы Executor и Participant также устаревшие

Особенности реализации

  • Сравнение списков исполнителей выполняется через SequenceEqual
  • Хеш-код вычисляется агрегацией хешей всех исполнителей
  • Реализация требует корректного переопределения Equals/GetHashCode в Executor
