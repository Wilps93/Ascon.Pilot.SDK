Класс Executor [Устаревший]

Описание

Представляет исполнителя задачи с настройками дедлайнов. Класс признан устаревшим - рекомендуется использовать современные альтернативы для новых разработок.

Наследуется от Participant, расширяя функционал управления сроками.

Наследование

Participant → Executor

Свойства

Свойство Тип Описание
Role string Роль исполнителя в задаче
IsDeadlineSet bool Установлен ли дедлайн для исполнителя
DeadlineDays int Количество дней на выполнение задачи (для относительных дедлайнов)
DeadlineTime DateTime Фиксированная дата дедлайна
IsRelativeDeadline bool Тип дедлайна: относительный (от даты начала) или фиксированный

Методы

Переопределенные методы

Equals(Executor) Сравнивает текущий объект с другим исполнителем
Equals(object) Сравнивает с произвольным объектом
GetHashCode() Возвращает хеш-код объекта

Примеры использования


// Создание исполнителя
var executor = new Executor
{
    Role = "developer",
    IsDeadlineSet = true,
    DeadlineDays = 5,
    IsRelativeDeadline = true
};

// Сравнение исполнителей
Executor another = GetExecutor();
bool areEqual = executor.Equals(another);

// Проверка дедлайна
if (executor.IsRelativeDeadline)
{
    DateTime deadline = DateTime.Now.AddDays(executor.DeadlineDays);
}
Важно:
  • Класс помечен как [Obsolete] - использование в новых проектах не рекомендуется
  • Сравнение учитывает все свойства, включая базовый класс Participant
  • Хеш-код зависит от всех полей, включая наследуемые
  • Для фиксированных дедлайнов используйте DeadlineTime вместо DeadlineDays

Особенности реализации

  • Сравнение объектов включает проверку базового класса Participant
  • Хеш-код вычисляется комбинацией всех свойств
  • Реализация учитывает возможное наследование через проверку GetType()
