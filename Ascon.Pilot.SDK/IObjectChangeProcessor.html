Интерфейс IObjectChangeProcessor

Назначение

Предоставляет механизм перехвата и обработки изменений объектов, вносимых текущим пользователем, с возможностью модификации или блокировки изменений.

Методы

Метод Параметры Возвращаемое значение Описание
ProcessChanges
  • changes: IEnumerable - Коллекция изменений
  • modifier: IObjectModifier - Объект для модификации данных
 bool Вызывается при сохранении изменений пользователем.
Возвращает true для применения изменений, false для отмены

Пример реализации


public class CustomChangeProcessor : IObjectChangeProcessor 
{
    public bool ProcessChanges(IEnumerable changes, IObjectModifier modifier)
    {
        foreach (var change in changes)
        {
            // Блокировка удаления объектов с определенным типом
            if (change.ChangeType == ChangeType.Deleted && change.Object.TypeId == 123)
            {
                return false; // Отмена операции
            }

            // Автоматическое заполнение атрибута при создании
            if (change.ChangeType == ChangeType.Created)
            {
                modifier.SetAttribute("AutoGeneratedField", DateTime.Now.ToString());
            }
        }
        return true;
    }
}

Ключевые особенности

  • Позволяет модифицировать изменения до их сохранения в БД
  • Может полностью отменять транзакцию, возвращая false
  • Работает в контексте текущего пользователя
  • Интегрируется с IObjectModifier для внесения дополнительных изменений
Важно:
  • Избегайте долгих операций в обработчике - это влияет на производительность
  • Не забывайте обрабатывать все типы изменений (Create/Update/Delete)
  • Изменения через modifier применяются только при возврате true
  • Для работы с файлами используйте методы modifier.AddFile() и др.
Предупреждение:
  • Некорректная обработка изменений может нарушить целостность данных
  • Не рекомендуется изменять объекты, не связанные с текущей транзакцией
